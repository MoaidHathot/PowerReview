1. INTRODUCTION                                  *power-review-introduction*

PowerReview.nvim is a Neovim plugin for reviewing Pull Requests from Azure
DevOps (with GitHub support planned) directly inside your editor. It supports
manual and LLM-assisted review with draft comment staging, rich UI panels,
diff views, and an MCP server for external AI agent integration.

==============================================================================
2. REQUIREMENTS                                  *power-review-requirements*

- Neovim >= 0.10.0
- nui.nvim (required)
- codediff.nvim (recommended, for diff views)
- neo-tree.nvim (optional, for file tree panel)
- telescope.nvim (optional, for fuzzy pickers)
- .NET 10 SDK (required, for the CLI tool)
- For Azure DevOps: `az` CLI or a Personal Access Token

==============================================================================
3. SETUP                                                *power-review-setup*

Minimal setup with lazy.nvim: >lua
  {
    "your-username/PowerReview.nvim",
    dependencies = {
      "MunifTanjim/nui.nvim",
      "esmuellert/codediff.nvim",
    },
    config = function()
      require("power-review").setup({})
    end,
  }
<

4. CONFIGURATION                                *power-review-configuration*

PowerReview uses a split configuration model:
- CLI config ($XDG_CONFIG_HOME/PowerReview/powerreview.json) owns auth, git
  strategy, provider settings, and data directory.
- Lua config (setup()) owns UI settings: keymaps, signs, panels, diff
  provider, and CLI executable path.

Call `require("power-review").setup(opts)` with any of these options:

                                                *power-review-config-cli*
cli ~
  CLI tool settings.

  `cli.executable`    Path or name of the CLI binary (default: "powerreview")

                                                 *power-review-config-ui*
ui ~
  UI component settings.

  `ui.files.provider`       "neo-tree" | "builtin" (default: "neo-tree")
  `ui.comments.float`       Float window: width, height, border
  `ui.comments.panel`       Panel: position, width, height
  `ui.comments.signs`       Sign characters: remote, draft, ai_draft
  `ui.diff.provider`        "native" | "codediff" (default: "native")

                                            *power-review-config-keymaps*
keymaps ~
  Global keymaps. Set any key to `false` to disable.

  `keymaps.open_review`      "<leader>pr"
  `keymaps.list_sessions`    "<leader>pl"
  `keymaps.toggle_files`     "<leader>pf"
  `keymaps.toggle_comments`  "<leader>pc"
  `keymaps.next_comment`     "]r"
  `keymaps.prev_comment`     "[r"
  `keymaps.add_comment`      "<leader>pa"
  `keymaps.reply_comment`    "<leader>pR"
  `keymaps.edit_comment`     "<leader>pe"
  `keymaps.approve_comment`  "<leader>pA"
  `keymaps.submit_all`       "<leader>pS"
  `keymaps.set_vote`         "<leader>pv"
  `keymaps.sync_threads`     "<leader>ps"
  `keymaps.close_review`     "<leader>pQ"
  `keymaps.delete_session`   "<leader>pD"

                                                 *power-review-config-log*
log ~
  `log.level`  "debug" | "info" | "warn" | "error" (default: "info")

                                        *power-review-config-cli-json*
CLI Configuration File ~
  The CLI reads `$XDG_CONFIG_HOME/PowerReview/powerreview.json` (or
  `%APPDATA%\PowerReview\powerreview.json` on Windows). Fields:

  `auth.azdo.method`        "auto" | "az_cli" | "pat" (default: "auto")
  `auth.azdo.pat_env_var`   Env var name for AzDO PAT (default: "AZDO_PAT")
  `auth.github.pat_env_var` Env var name for GitHub token (default: "GITHUB_TOKEN")
  `git.strategy`            "worktree" | "checkout" (default: "worktree")
  `git.worktree_dir`        Relative directory name (default: ".power-review-worktrees")
  `git.cleanup_on_close`    Remove worktree on close (default: true)
  `data_dir`                Override default data directory for sessions

5. COMMANDS                                          *power-review-commands*

All commands use the `:PowerReview` prefix.

                                                       *:PowerReview-open*
:PowerReview open [url]
  Start a review from a PR URL, or pick from saved sessions if no URL.

:PowerReview {url}
  Shorthand: start a review directly from a URL.

                                                   *:PowerReview-sessions*
:PowerReview sessions
  Toggle the session management panel. Shows all saved sessions with
  resume, delete, and new review options.

                                                       *:PowerReview-list*
:PowerReview list
  Print all saved session summaries to messages.

                                                     *:PowerReview-delete*
:PowerReview delete [session_id]
  Delete a saved session. Opens a picker if no ID provided.

                                                      *:PowerReview-clean*
:PowerReview clean
  Delete all saved sessions (with confirmation).

                                                      *:PowerReview-files*
:PowerReview files
  Toggle the changed files panel (Neo-tree or builtin).

                                                   *:PowerReview-comments*
:PowerReview comments
  Toggle the all-comments split panel.

                                                     *:PowerReview-drafts*
:PowerReview drafts
  Toggle the draft management panel.

                                                    *:PowerReview-comment*
:PowerReview comment
  Add a comment at the cursor position. Supports visual mode selection
  for range comments (use `:'<,'>PowerReview comment`).

                                                     *:PowerReview-thread*
:PowerReview thread
  View and reply to the comment thread at the cursor position.

                                                       *:PowerReview-diff*
:PowerReview diff [file_path]
  Open the diff view. If a file path is given, opens that file's diff.
  Otherwise opens the full diff explorer.

                                                       *:PowerReview-next*
:PowerReview next
  Jump to the next comment sign in the current buffer.

                                                       *:PowerReview-prev*
:PowerReview prev
  Jump to the previous comment sign in the current buffer.

                                                    *:PowerReview-approve*
:PowerReview approve [draft_id]
  Approve a draft comment. Operates on cursor position if no ID given.

                                                *:PowerReview-approve_all*
:PowerReview approve_all
  Approve all draft comments (with confirmation).

                                                     *:PowerReview-submit*
:PowerReview submit
  Submit all pending comments to the remote provider. Shows progress and
  offers retry on failure.

                                                       *:PowerReview-vote*
:PowerReview vote
  Open vote selection UI. Options: Approve, Approve with suggestions,
  No vote, Wait for author, Reject.

                                                     *:PowerReview-refresh*
:PowerReview refresh
  Re-fetch PR data (changed files) from the remote provider.

                                                        *:PowerReview-sync*
:PowerReview sync
  Sync remote comment threads from the provider.

                                                      *:PowerReview-close*
:PowerReview close
  Close the current review session. Saves state and optionally cleans
  up the git worktree.

7. LUA API                                                *power-review-api*

The public API is at `require("power-review").api`:

                                         *power-review-api.get_changed_files*
api.get_changed_files()
  Returns: `ChangedFile[]|nil, string|nil`

                                            *power-review-api.get_file_diff*
api.get_file_diff({file_path})
  Returns: `string|nil, string|nil`

                                          *power-review-api.get_all_threads*
api.get_all_threads()
  Returns: `table|nil, string|nil`

                                    *power-review-api.get_threads_for_file*
api.get_threads_for_file({file_path})
  Returns: `table|nil, string|nil`

                                     *power-review-api.create_draft_comment*
api.create_draft_comment({opts})
  Parameters:
    {opts.file_path}   string   Relative file path
    {opts.line_start}  number   1-indexed line
    {opts.line_end}    number   Optional end line for range
    {opts.body}        string   Markdown content
    {opts.author}      string   "user" | "ai" (default: "user")
  Returns: `DraftComment|nil, string|nil`

                                       *power-review-api.edit_draft_comment*
api.edit_draft_comment({draft_id}, {new_body})
  Only works on drafts with status == "draft".
  Returns: `boolean, string|nil`

                                     *power-review-api.delete_draft_comment*
api.delete_draft_comment({draft_id})
  Only works on drafts with status == "draft".
  Returns: `boolean, string|nil`

                                           *power-review-api.approve_draft*
api.approve_draft({draft_id})
  Move draft from "draft" to "pending".
  Returns: `boolean, string|nil`

                                       *power-review-api.approve_all_drafts*
api.approve_all_drafts()
  Returns: `number` (count of approved)

                                         *power-review-api.unapprove_draft*
api.unapprove_draft({draft_id})
  Revert from "pending" back to "draft".
  Returns: `boolean, string|nil`

                                           *power-review-api.submit_pending*
api.submit_pending({callback}, {progress_cb?})
  Async. Submits all pending comments to remote.

                                  *power-review-api.retry_failed_submissions*
api.retry_failed_submissions({failed_drafts}, {callback})
  Async. Retries previously failed submissions.

                                               *power-review-api.set_vote*
api.set_vote({vote}, {callback})
  Async. Vote values: 10 (approve), 5 (approve w/ suggestions),
  0 (no vote), -5 (wait), -10 (reject).

                                        *power-review-api.get_review_session*
api.get_review_session()
  Returns session metadata table or nil.

                                            *power-review-api.sync_threads*
api.sync_threads({callback})
  Async. Fetches the latest remote comment threads from the provider
  and updates the local session state.

                                            *power-review-api.close_review*
api.close_review({callback?})
  Async. Closes the current review session. Saves state and tears
  down UI. Optionally cleans up the git worktree.

                                          *power-review-api.reply_to_thread*
api.reply_to_thread({opts})
  Create a draft reply to an existing thread.
  Returns: `DraftComment|nil, string|nil`

8. MCP SERVER                                            *power-review-mcp*

The CLI tool doubles as an MCP (Model Context Protocol) server. Running
`powerreview mcp` starts a stdio-based MCP server that AI agents can
connect to directly -- no Neovim instance required.

Configure your AI tool's MCP settings: >json
  {
    "mcpServers": {
      "power-review": {
        "command": "powerreview",
        "args": ["mcp"]
      }
    }
  }
<

MCP tools: GetReviewSession, ListChangedFiles, GetFileDiff,
ListCommentThreads, CreateComment, ReplyToThread,
EditDraftComment, DeleteDraftComment.

AI-created comments are tagged with author=ai and start as drafts
that require user approval before submission.

10. DRAFT WORKFLOW                                    *power-review-drafts*

Comments follow a strict lifecycle: draft -> pending -> submitted.

  draft       Created locally. Editable/deletable. LLMs can modify.
  pending     Approved, ready to submit. Immutable to LLMs.
  submitted   Sent to remote. Immutable.

This ensures LLM-generated comments always go through human approval.

11. SESSION STATE FILE                            *power-review-state-file*

Each review session is persisted as a JSON file on disk by the CLI tool.
This section documents the file location and schema.

FILE LOCATION ~
                                                  *power-review-state-path*
Session files are managed by the CLI and stored at:
>
  {data_dir}/sessions/{session_id}.json
<
The data directory is resolved in this order:
1. `data_dir` field in CLI's `powerreview.json`
2. `$XDG_DATA_HOME/PowerReview`
3. Windows: `%LOCALAPPDATA%\PowerReview`; Linux/Mac: `~/.local/share/PowerReview`

Run `powerreview config --path-only` to see the resolved config path.

The `session_id` is constructed as `{org}_{project}_{repo}_{pr_id}`, with
non-alphanumeric, non-hyphen characters replaced by underscores.
Example: `my-org_my-project_my-repo_42.json`

The CLI outputs v3 session format (nested JSON). The Neovim plugin's
`cli.adapt_session()` converts this to a flat v2-compatible shape that
all UI code expects. See `lua/power-review/cli.lua` for the adapter.

See: `doc/session-schema.md` for the full schema specification.

SCHEMA OVERVIEW ~
                                                *power-review-state-schema*
The CLI outputs v3 JSON format with a nested structure (pull_request,
provider, git groupings). The full schema is documented in
`doc/session-schema.md`.

The Neovim plugin adapter converts v3 to a flat v2-compatible shape:
  - `pull_request.id` -> `pr_id`
  - `pull_request.title` -> `pr_title`
  - `provider.type` -> `provider_type`
  - `git.worktree_path` -> `worktree_path`
  - `threads.items` -> `threads` (array)
  - `drafts` map `{uuid: draft}` -> `drafts` (sorted array)
  - `vote` string enum -> numeric value

See `lua/power-review/cli.lua:adapt_session()` for the full mapping.

PERSISTENCE BEHAVIOR ~
                                          *power-review-state-persistence*

The CLI re-saves the session to disk after every state-changing operation:

  - Adding, editing, or deleting a draft comment
  - Approving or unapproving a draft
  - Submitting pending comments
  - Setting a vote
  - Syncing remote threads
  - Opening, refreshing, or closing a session

After any CLI mutation, the Neovim plugin reloads the session from
the CLI using `powerreview session --pr-url ...`.

SESSION SUMMARIES ~
                                             *power-review-state-summary*

When listing sessions (`:PowerReview list`, `:PowerReview sessions`),
the CLI returns lighter summary projections via `powerreview sessions list`.
