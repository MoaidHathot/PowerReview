1. INTRODUCTION                                  *power-review-introduction*

PowerReview.nvim is a Neovim plugin for reviewing Pull Requests from Azure
DevOps (with GitHub support planned) directly inside your editor. It supports
manual and LLM-assisted review with draft comment staging, rich UI panels,
diff views, and an MCP server for external AI agent integration.

==============================================================================
2. REQUIREMENTS                                  *power-review-requirements*

- Neovim >= 0.10.0
- nui.nvim (required)
- codediff.nvim (recommended, for diff views)
- neo-tree.nvim (optional, for file tree panel)
- telescope.nvim (optional, for fuzzy pickers)
- `curl` on PATH
- For Azure DevOps: `az` CLI or a Personal Access Token

==============================================================================
3. SETUP                                                *power-review-setup*

Minimal setup with lazy.nvim: >lua
  {
    "your-username/PowerReview.nvim",
    dependencies = {
      "MunifTanjim/nui.nvim",
      "esmuellert/codediff.nvim",
    },
    config = function()
      require("power-review").setup({})
    end,
  }
<

4. CONFIGURATION                                *power-review-configuration*

Call `require("power-review").setup(opts)` with any of these options:

                                              *power-review-config-repos*
repos ~
  Table keyed by absolute repo path. Maps repositories to provider configs.
  Example: >lua
    repos = {
      ["/path/to/repo"] = {
        provider = "azdo",
        azdo = {
          organization = "myorg",
          project = "myproject",
          repository = "myrepo",
        },
      },
    }
<

                                               *power-review-config-auth*
auth ~
  Authentication settings per provider.

  `auth.azdo.method`    "auto" | "az_cli" | "pat" (default: "auto")
  `auth.azdo.pat`       PAT string, or use env: POWER_REVIEW_AZDO_PAT
  `auth.github.pat`     PAT string, or use env: GITHUB_TOKEN

                                                *power-review-config-git*
git ~
  Git strategy settings.

  `git.strategy`          "worktree" | "checkout" (default: "worktree")
  `git.worktree_dir`      Relative directory name (default: ".power-review-worktrees")
  `git.cleanup_on_close`  Remove worktree on close (default: true)

                                                 *power-review-config-ui*
ui ~
  UI component settings.

  `ui.files.provider`       "neo-tree" | "builtin" (default: "neo-tree")
  `ui.comments.float`       Float window: width, height, border
  `ui.comments.panel`       Panel: position, width, height
  `ui.comments.signs`       Sign characters: remote, draft, ai_draft
  `ui.diff.provider`        "native" | "codediff" (default: "native")

                                            *power-review-config-keymaps*
keymaps ~
  Global keymaps. Set any key to `false` to disable.

  `keymaps.open_review`      "<leader>pr"
  `keymaps.list_sessions`    "<leader>pl"
  `keymaps.toggle_files`     "<leader>pf"
  `keymaps.toggle_comments`  "<leader>pc"
  `keymaps.next_comment`     "]r"
  `keymaps.prev_comment`     "[r"
  `keymaps.add_comment`      "<leader>pa"
  `keymaps.reply_comment`    "<leader>pR"
  `keymaps.edit_comment`     "<leader>pe"
  `keymaps.approve_comment`  "<leader>pA"
  `keymaps.submit_all`       "<leader>pS"
  `keymaps.set_vote`         "<leader>pv"
  `keymaps.sync_threads`     "<leader>ps"
  `keymaps.close_review`     "<leader>pQ"
  `keymaps.delete_session`   "<leader>pD"

                                                *power-review-config-mcp*
mcp ~
  MCP server integration.

  `mcp.enabled`  Write server_info.json for MCP server (default: false)

                                                *power-review-config-log*
log ~
  `log.level`  "debug" | "info" | "warn" | "error" (default: "info")

5. COMMANDS                                          *power-review-commands*

All commands use the `:PowerReview` prefix.

                                                       *:PowerReview-open*
:PowerReview open [url]
  Start a review from a PR URL, or pick from saved sessions if no URL.

:PowerReview {url}
  Shorthand: start a review directly from a URL.

                                                   *:PowerReview-sessions*
:PowerReview sessions
  Toggle the session management panel. Shows all saved sessions with
  resume, delete, and new review options.

                                                       *:PowerReview-list*
:PowerReview list
  Print all saved session summaries to messages.

                                                     *:PowerReview-delete*
:PowerReview delete [session_id]
  Delete a saved session. Opens a picker if no ID provided.

                                                      *:PowerReview-clean*
:PowerReview clean
  Delete all saved sessions (with confirmation).

                                                      *:PowerReview-files*
:PowerReview files
  Toggle the changed files panel (Neo-tree or builtin).

                                                   *:PowerReview-comments*
:PowerReview comments
  Toggle the all-comments split panel.

                                                     *:PowerReview-drafts*
:PowerReview drafts
  Toggle the draft management panel.

                                                    *:PowerReview-comment*
:PowerReview comment
  Add a comment at the cursor position. Supports visual mode selection
  for range comments (use `:'<,'>PowerReview comment`).

                                                     *:PowerReview-thread*
:PowerReview thread
  View and reply to the comment thread at the cursor position.

                                                       *:PowerReview-diff*
:PowerReview diff [file_path]
  Open the diff view. If a file path is given, opens that file's diff.
  Otherwise opens the full diff explorer.

                                                       *:PowerReview-next*
:PowerReview next
  Jump to the next comment sign in the current buffer.

                                                       *:PowerReview-prev*
:PowerReview prev
  Jump to the previous comment sign in the current buffer.

                                                    *:PowerReview-approve*
:PowerReview approve [draft_id]
  Approve a draft comment. Operates on cursor position if no ID given.

                                                *:PowerReview-approve_all*
:PowerReview approve_all
  Approve all draft comments (with confirmation).

                                                     *:PowerReview-submit*
:PowerReview submit
  Submit all pending comments to the remote provider. Shows progress and
  offers retry on failure.

                                                       *:PowerReview-vote*
:PowerReview vote
  Open vote selection UI. Options: Approve, Approve with suggestions,
  No vote, Wait for author, Reject.

                                                     *:PowerReview-refresh*
:PowerReview refresh
  Re-fetch PR data (changed files) from the remote provider.

                                                        *:PowerReview-sync*
:PowerReview sync
  Sync remote comment threads from the provider.

                                                      *:PowerReview-close*
:PowerReview close
  Close the current review session. Saves state and optionally cleans
  up the git worktree.

7. LUA API                                                *power-review-api*

The public API is at `require("power-review").api`:

                                         *power-review-api.get_changed_files*
api.get_changed_files()
  Returns: `ChangedFile[]|nil, string|nil`

                                            *power-review-api.get_file_diff*
api.get_file_diff({file_path})
  Returns: `string|nil, string|nil`

                                          *power-review-api.get_all_threads*
api.get_all_threads()
  Returns: `table|nil, string|nil`

                                    *power-review-api.get_threads_for_file*
api.get_threads_for_file({file_path})
  Returns: `table|nil, string|nil`

                                     *power-review-api.create_draft_comment*
api.create_draft_comment({opts})
  Parameters:
    {opts.file_path}   string   Relative file path
    {opts.line_start}  number   1-indexed line
    {opts.line_end}    number   Optional end line for range
    {opts.body}        string   Markdown content
    {opts.author}      string   "user" | "ai" (default: "user")
  Returns: `DraftComment|nil, string|nil`

                                       *power-review-api.edit_draft_comment*
api.edit_draft_comment({draft_id}, {new_body})
  Only works on drafts with status == "draft".
  Returns: `boolean, string|nil`

                                     *power-review-api.delete_draft_comment*
api.delete_draft_comment({draft_id})
  Only works on drafts with status == "draft".
  Returns: `boolean, string|nil`

                                           *power-review-api.approve_draft*
api.approve_draft({draft_id})
  Move draft from "draft" to "pending".
  Returns: `boolean, string|nil`

                                       *power-review-api.approve_all_drafts*
api.approve_all_drafts()
  Returns: `number` (count of approved)

                                         *power-review-api.unapprove_draft*
api.unapprove_draft({draft_id})
  Revert from "pending" back to "draft".
  Returns: `boolean, string|nil`

                                           *power-review-api.submit_pending*
api.submit_pending({callback}, {progress_cb?})
  Async. Submits all pending comments to remote.

                                  *power-review-api.retry_failed_submissions*
api.retry_failed_submissions({failed_drafts}, {callback})
  Async. Retries previously failed submissions.

                                               *power-review-api.set_vote*
api.set_vote({vote}, {callback})
  Async. Vote values: 10 (approve), 5 (approve w/ suggestions),
  0 (no vote), -5 (wait), -10 (reject).

                                        *power-review-api.get_review_session*
api.get_review_session()
  Returns session metadata table or nil.

                                            *power-review-api.sync_threads*
api.sync_threads({callback})
  Async. Fetches the latest remote comment threads from the provider
  and updates the local session state.

                                            *power-review-api.close_review*
api.close_review({callback?})
  Async. Closes the current review session. Saves state and tears
  down UI. Optionally cleans up the git worktree.

                                          *power-review-api.reply_to_thread*
api.reply_to_thread({opts})
  Create a draft reply to an existing thread.
  Returns: `DraftComment|nil, string|nil`

8. MCP SERVER                                            *power-review-mcp*

PowerReview includes a TypeScript MCP server in the `mcp/` directory. It
connects to Neovim via msgpack-RPC and exposes review tools to AI agents.

Enable with `mcp.enabled = true` in setup. See README for full setup
instructions and `.mcp.json` configuration.

MCP tools: get_review_session, list_changed_files, get_file_diff,
list_comment_threads, create_comment, reply_to_thread,
edit_draft_comment, delete_draft_comment.

10. DRAFT WORKFLOW                                    *power-review-drafts*

Comments follow a strict lifecycle: draft -> pending -> submitted.

  draft       Created locally. Editable/deletable. LLMs can modify.
  pending     Approved, ready to submit. Immutable to LLMs.
  submitted   Sent to remote. Immutable.

This ensures LLM-generated comments always go through human approval.

11. SESSION STATE FILE                            *power-review-state-file*

Each review session is persisted as a JSON file on disk. This section
documents the file location and full schema.

FILE LOCATION ~
                                                  *power-review-state-path*
Session files are stored at:
>
  {vim.fn.stdpath("data")}/power-review/sessions/{session_id}.json
<
Typical paths:
- Linux/macOS: `~/.local/share/nvim/power-review/sessions/`
- Windows:     `%LOCALAPPDATA%/nvim-data/power-review/sessions/`

The `session_id` is constructed as `{org}_{project}_{repo}_{pr_id}`, with
non-alphanumeric, non-hyphen characters replaced by underscores.
Example: `my-org_my-project_my-repo_42.json`

Writes are atomic: data is written to a `.tmp` file first, then renamed
to the final `.json` path.

See: `lua/power-review/store/json.lua`
     `lua/power-review/store/init.lua`

SCHEMA OVERVIEW ~
                                                *power-review-state-schema*
The JSON file serializes a `PowerReview.ReviewSession` table. The
authoritative type definition is in `lua/power-review/types.lua:88-109`.

Top-level structure:
>json
  {
    "version":        1,
    "id":             "org_project_repo_42",
    "pr_id":          42,
    "provider_type":  "azdo",
    "org":            "my-org",
    "project":        "my-project",
    "repo":           "my-repo",
    "pr_url":         "https://dev.azure.com/...",
    "pr_title":       "Add new feature",
    "pr_description": "This PR adds...",
    "pr_author":      "John Doe",
    "source_branch":  "feature/foo",
    "target_branch":  "main",
    "worktree_path":  "/path/to/worktree",
    "git_strategy":   "worktree",
    "created_at":     "2026-03-27T10:00:00Z",
    "updated_at":     "2026-03-27T12:30:00Z",
    "vote":           10,
    "files":          [ ... ],
    "threads":        [ ... ],
    "drafts":         [ ... ]
  }
<

SESSION METADATA FIELDS ~
                                              *power-review-state-metadata*

  `version`           number
    Schema version for migration safety. Currently always `1`.
    Defaulted to `1` on load if missing.

  `id`                string
    Unique session identifier: `{org}_{project}_{repo}_{pr_id}`.
    Also used as the JSON filename (with `.json` appended).

  `pr_id`             number
    Pull request number/ID from the provider.

  `provider_type`     string
    Provider type: `"azdo"` (Azure DevOps) or `"github"`.

  `org`               string
    Organization name (AzDO) or repository owner (GitHub).

  `project`           string
    Project name (AzDO) or repository name (GitHub).

  `repo`              string
    Repository name.

PR METADATA FIELDS ~
                                           *power-review-state-pr-metadata*

  `pr_url`            string
    The original URL used to open the review.

  `pr_title`          string
    Pull request title.

  `pr_description`    string
    Pull request description body (markdown).

  `pr_author`         string
    Display name of the PR author.

  `source_branch`     string
    Source/feature branch of the PR.

  `target_branch`     string
    Target/base branch of the PR.

GIT & WORKSPACE FIELDS ~
                                                  *power-review-state-git*

  `worktree_path`     string | null
    Filesystem path to the git worktree created for the review.
    `null` if no worktree was created.

  `git_strategy`      string
    How git was set up for the review:
    - `"worktree"`:  Separate worktree created for the source branch.
    - `"checkout"`:  Branch checkout in the main repository.

TIMESTAMP FIELDS ~
                                           *power-review-state-timestamps*

  `created_at`        string
    ISO 8601 UTC timestamp (`YYYY-MM-DDTHH:MM:SSZ`) of session creation.

  `updated_at`        string
    ISO 8601 UTC timestamp, refreshed on every save/mutation.

REVIEW VOTE FIELD ~
                                                 *power-review-state-vote*

  `vote`              number | null
    The user's review vote. `null` means not yet voted.
    AzDO vote values:
      `10`    Approved
      `5`     Approved with suggestions
      `0`     No vote / reset
      `-5`    Wait for author
      `-10`   Rejected

CHANGED FILES ARRAY ~
                                                *power-review-state-files*

  `files`             array of ChangedFile objects.
    Each element describes a file changed in the PR:

    `path`              string
      Relative file path.

    `original_path`     string | null
      Original path before rename. Only present for renames.

    `change_type`       string
      One of: `"add"`, `"edit"`, `"delete"`, `"rename"`.

    `additions`         number | null
      Lines added. Provider-dependent; may be absent.

    `deletions`         number | null
      Lines deleted. Provider-dependent; may be absent.

  Example:
>json
    {
      "path": "src/utils/helper.lua",
      "original_path": null,
      "change_type": "edit",
      "additions": 15,
      "deletions": 3
    }
<

REMOTE COMMENT THREADS ARRAY ~
                                              *power-review-state-threads*

  `threads`           array of CommentThread objects.
    Threads fetched from the provider and cached locally.

    `id`                number
      Provider-assigned thread ID.

    `file_path`         string | null
      Relative file path. `null` for PR-level threads.

    `line_start`        number | null
      Start line number of the code range.

    `line_end`          number | null
      End line number of the code range.

    `col_start`         number | null
      1-indexed start column offset. Set only if provider reports
      offset > 1.

    `col_end`           number | null
      1-indexed end column offset. Set only if provider reports
      offset > 1.

    `status`            string
      Thread resolution status. One of: `"active"`, `"fixed"`,
      `"wontfix"`, `"closed"`, `"bydesign"`, `"pending"`.

    `is_deleted`        boolean
      Whether the thread has been soft-deleted.

    `comments`          array of Comment objects (see below).

  Each Comment within a thread:

    `id`                number
      Provider-assigned comment ID.

    `thread_id`         number
      Parent thread ID.

    `author`            string
      Comment author display name.

    `body`              string
      Comment content (markdown).

    `created_at`        string
      ISO 8601 timestamp of creation.

    `updated_at`        string
      ISO 8601 timestamp of last update.

    `is_deleted`        boolean
      Whether the comment has been soft-deleted.

  Example:
>json
    {
      "id": 12345,
      "file_path": "src/main.lua",
      "line_start": 42,
      "line_end": 45,
      "col_start": null,
      "col_end": null,
      "status": "active",
      "is_deleted": false,
      "comments": [
        {
          "id": 67890,
          "thread_id": 12345,
          "author": "Jane Smith",
          "body": "Consider using pcall here.",
          "created_at": "2026-03-26T08:00:00Z",
          "updated_at": "2026-03-26T09:15:00Z",
          "is_deleted": false
        }
      ]
    }
<

LOCAL DRAFT COMMENTS ARRAY ~
                                               *power-review-state-drafts*

  `drafts`            array of DraftComment objects.
    Locally-created comments not yet submitted to the provider.
    See |power-review-drafts| for the draft lifecycle.

    `id`                string
      Locally-generated UUID v4.

    `file_path`         string
      Relative file path the comment targets.

    `line_start`        number
      1-indexed start line number.

    `line_end`          number | null
      End line for multi-line range comments.

    `col_start`         number | null
      1-indexed start column (byte offset within `line_start`).

    `col_end`           number | null
      1-indexed end column (byte offset within `line_end` or
      `line_start`).

    `body`              string
      Comment content (markdown).

    `status`            string
      Draft lifecycle status:
      - `"draft"`:     Editable. Can be modified or deleted.
      - `"pending"`:   Approved, ready for submission. Immutable.
      - `"submitted"`: Sent to the remote provider. Immutable.

    `author`            string
      Who created the draft: `"user"` or `"ai"` (LLM-generated).

    `thread_id`         number | null
      `null` for new standalone threads. Set to a remote thread ID
      when this draft is a reply to an existing thread.

    `parent_comment_id` number | null
      For replies, the specific comment being replied to.

    `created_at`        string
      ISO 8601 UTC timestamp.

    `updated_at`        string
      ISO 8601 UTC timestamp.

  Example:
>json
    {
      "id": "a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d",
      "file_path": "src/main.lua",
      "line_start": 42,
      "line_end": null,
      "col_start": null,
      "col_end": null,
      "body": "This should be refactored",
      "status": "draft",
      "author": "user",
      "thread_id": null,
      "parent_comment_id": null,
      "created_at": "2026-03-27T11:00:00Z",
      "updated_at": "2026-03-27T11:30:00Z"
    }
<

PERSISTENCE BEHAVIOR ~
                                          *power-review-state-persistence*

The session is re-saved to disk after every state-changing operation:

  - Adding, editing, or deleting a draft comment
  - Approving or unapproving a draft
  - Submitting pending comments
  - Retrying failed submissions
  - Setting a vote
  - Syncing remote threads
  - Refreshing the session
  - Resuming or closing a session

On load, migration defaults are applied for backward compatibility:
  - `drafts` defaults to `{}`
  - `files` defaults to `{}`
  - `version` defaults to `1`

SESSION SUMMARIES ~
                                             *power-review-state-summary*

When listing sessions (`:PowerReview list`, `:PowerReview sessions`),
a lighter `PowerReview.SessionSummary` projection is returned instead
of the full session. It contains:

  `id`, `pr_id`, `pr_title`, `pr_url`, `provider_type`, `org`,
  `project`, `repo`, `draft_count`, `created_at`, `updated_at`

See: `lua/power-review/store/init.lua:50-81`
