1. INTRODUCTION                                  *power-review-introduction*

PowerReview.nvim is a Neovim plugin for reviewing Pull Requests from Azure
DevOps (with GitHub support planned) directly inside your editor. It supports
manual and LLM-assisted review with draft comment staging, rich UI panels,
diff views, and an MCP server for external AI agent integration.

==============================================================================
2. REQUIREMENTS                                  *power-review-requirements*

- Neovim >= 0.10.0
- nui.nvim (recommended for full UI; falls back to vim.ui.select/quickfix)
- neo-tree.nvim (optional, for file tree panel)
- telescope.nvim (optional, for fuzzy pickers)
- fzf-lua (optional, alternative fuzzy pickers)
- codediff.nvim (optional, alternative diff provider)
- nvim-web-devicons (optional, for file type icons)
- .NET 10 SDK (required, for the CLI tool)
- For Azure DevOps: `az` CLI or a Personal Access Token

Run `:checkhealth power-review` to verify your setup.

3. SETUP                                                *power-review-setup*

Minimal setup with lazy.nvim: >lua
  {
    "your-username/PowerReview.nvim",
    dependencies = {
      "MunifTanjim/nui.nvim",
    },
    config = function()
      require("power-review").setup({})
    end,
  }
<

4. CONFIGURATION                                *power-review-configuration*

PowerReview uses a split configuration model:
- CLI config ($XDG_CONFIG_HOME/PowerReview/powerreview.json) owns auth, git
  strategy, provider settings, and data directory.
- Lua config (setup()) owns UI settings: keymaps, signs, panels, diff
  provider, and CLI executable path.

Call `require("power-review").setup(opts)` with any of these options:

                                                *power-review-config-cli*
cli ~
  CLI tool settings.

  `cli.executable`    Table or string for CLI invocation.
                      Default: { "dnx", "--yes", "--add-source",
                        "https://api.nuget.org/v3/index.json",
                        "PowerReview", "--" }
  `cli.timeouts`      Table of operation timeouts (ms).
                      default=30000, open=60000, submit=60000,
                      vote=30000, sync=30000

                                                 *power-review-config-ui*
ui ~
  UI component settings.

  `ui.files.provider`       "neo-tree" | "builtin" (default: "neo-tree")
  `ui.pickers.provider`     "telescope" | "fzf-lua" (default: "telescope")
  `ui.comments.float`       Float window: width, height, border
  `ui.comments.panel`       Panel: position, width, height
  `ui.comments.signs`       Sign characters: remote, draft, ai_draft
  `ui.comments.preview_debounce`  Ms delay for live preview (default: 150)
  `ui.diff.provider`        "native" | "codediff" (default: "native")
  `ui.virtual_text.max_length`  Max inline preview length (default: 80)
  `ui.flash.duration`       Flash highlight duration ms (default: 2000)
  `ui.colors`               Colors table (undercurl, flash, diff, statusline)

                                           *power-review-config-watcher*
watcher ~
  `watcher.enabled`       Enable session file watching (default: true)
  `watcher.debounce_ms`   Debounce delay (default: 200)

                                     *power-review-config-notifications*
notifications ~
  `notifications.enabled`        Master toggle (default: true)
  `notifications.ai_activity`    AI draft changes (default: true)
  `notifications.sync_complete`  Thread sync completion (default: true)
  `notifications.watcher`        External session changes (default: true)

                                                 *power-review-config-log*
log ~
  `log.level`  "debug" | "info" | "warn" | "error" (default: "info")

                                        *power-review-config-cli-json*
CLI Configuration File ~
  The CLI reads `$XDG_CONFIG_HOME/PowerReview/powerreview.json` (or
  `%APPDATA%\PowerReview\powerreview.json` on Windows). Fields:

  `auth.azdo.method`        "auto" | "az_cli" | "pat" (default: "auto")
  `auth.azdo.pat_env_var`   Env var name for AzDO PAT (default: "AZDO_PAT")
  `auth.github.pat_env_var` Env var name for GitHub token (default: "GITHUB_TOKEN")
  `git.strategy`            "worktree" | "checkout" (default: "worktree")
  `git.worktree_dir`        Directory hosting review worktrees. Relative
                            paths (default: ".power-review-worktrees") are
                            joined with the repo root. Absolute paths place
                            worktrees outside the repo, namespaced by repo
                            identity so multiple repos can share one base.
  `git.always_separate_worktree`
                            If true, always create a separate linked
                            worktree even when the main repo is already on
                            the PR's source branch. Keeps the main repo's
                            branch state untouched (default: false).
  `git.cleanup_on_close`    Remove worktree on close (default: true)
  `data_dir`                Override default data directory for sessions

5. COMMANDS                                          *power-review-commands*

All commands use the `:PowerReview` prefix.

                                                       *:PowerReview-open*
:PowerReview open [url]
  Start a review from a PR URL, or pick from saved sessions if no URL.

:PowerReview {url}
  Shorthand: start a review directly from a URL.

                                                   *:PowerReview-sessions*
:PowerReview sessions
  Toggle the session management panel.

                                                       *:PowerReview-list*
:PowerReview list
  Print all saved session summaries to messages.

                                                     *:PowerReview-delete*
:PowerReview delete [session_id]
  Delete a saved session. Opens a picker if no ID provided.

                                                      *:PowerReview-clean*
:PowerReview clean
  Delete all saved sessions (with confirmation).

                                                      *:PowerReview-files*
:PowerReview files
  Toggle the changed files panel (Neo-tree or builtin).

                                                   *:PowerReview-comments*
:PowerReview comments
  Toggle the all-comments split panel.

                                                     *:PowerReview-drafts*
:PowerReview drafts
  Toggle the draft management panel.

                                                    *:PowerReview-comment*
:PowerReview comment
  Add a comment at the cursor position. Supports visual mode selection
  for range comments (use `:'<,'>PowerReview comment`).

                                                     *:PowerReview-thread*
:PowerReview thread
  View and reply to the comment thread at the cursor position.

                                                       *:PowerReview-diff*
:PowerReview diff [file_path]
  Open the diff view for a file. Opens the explorer if no path given.

                                                       *:PowerReview-next*
:PowerReview next
  Jump to the next comment sign in the current buffer.

                                                       *:PowerReview-prev*
:PowerReview prev
  Jump to the previous comment sign in the current buffer.

                                                    *:PowerReview-approve*
:PowerReview approve [draft_id]
  Approve a draft comment. Operates on cursor position if no ID given.

                                                *:PowerReview-approve_all*
:PowerReview approve_all
  Approve all draft comments (with confirmation).

                                                     *:PowerReview-submit*
:PowerReview submit
  Submit all pending comments to the remote provider.

                                                       *:PowerReview-vote*
:PowerReview vote
  Open vote selection UI.

                                                    *:PowerReview-refresh*
:PowerReview refresh
  Re-fetch PR data from the remote provider.

                                                       *:PowerReview-sync*
:PowerReview sync
  Sync remote comment threads from the provider.

                                                      *:PowerReview-close*
:PowerReview close
  Close the current review session.

                                             *:PowerReview-show_description*
:PowerReview show_description
  Toggle the PR description floating viewer.

                                            *:PowerReview-resolve_thread*
:PowerReview resolve_thread <thread_id> <status>
  Set thread status. Statuses: active, fixed, wontfix, closed,
  bydesign, pending.

                                         *:PowerReview-toggle_notifications*
:PowerReview toggle_notifications
  Toggle the notification system on/off.

                                            *:PowerReview-mark_reviewed*
:PowerReview mark_reviewed [file_path]
  Mark a file as reviewed. Resolves from current buffer if omitted.

                                          *:PowerReview-unmark_reviewed*
:PowerReview unmark_reviewed [file_path]
  Unmark a file as reviewed.

                                        *:PowerReview-mark_all_reviewed*
:PowerReview mark_all_reviewed
  Mark all changed files as reviewed.

                                          *:PowerReview-check_iteration*
:PowerReview check_iteration
  Check for new PR iterations from remote.

                                           *:PowerReview-iteration_diff*
:PowerReview iteration_diff [file_path]
  Show diff between last-reviewed and current iteration for a file.

                                            *power-review-api.get_file_diff*
api.get_file_diff({file_path})
  Returns: `string|nil, string|nil`

                                          *power-review-api.get_all_threads*
api.get_all_threads()
  Returns: `table|nil, string|nil`

                                    *power-review-api.get_threads_for_file*
api.get_threads_for_file({file_path})
  Returns: `table|nil, string|nil`

                                     *power-review-api.create_draft_comment*
api.create_draft_comment({opts})
  Parameters:
    {opts.file_path}   string   Relative file path
    {opts.line_start}  number   1-indexed line
    {opts.line_end}    number   Optional end line for range
    {opts.col_start}   number   Optional start column
    {opts.col_end}     number   Optional end column

  Only works on drafts with status == "draft".
  Returns: `boolean, string|nil`

                                     *power-review-api.delete_draft_comment*
api.delete_draft_comment({draft_id})
  Only works on drafts with status == "draft".
  Returns: `boolean, string|nil`

                                           *power-review-api.approve_draft*
api.approve_draft({draft_id})
  Move draft from "draft" to "pending".
  Returns: `boolean, string|nil`

                                       *power-review-api.approve_all_drafts*
api.approve_all_drafts()
  Returns: `number` (count of approved)

                                         *power-review-api.unapprove_draft*
api.unapprove_draft({draft_id})
  Revert from "pending" back to "draft".
  Returns: `boolean, string|nil`

                                           *power-review-api.submit_pending*
api.submit_pending({callback}, {progress_cb?})
  Async. Submits all pending comments to remote.

                                  *power-review-api.retry_failed_submissions*
api.retry_failed_submissions({failed_drafts}, {callback})
  Async. Retries previously failed submissions.

                                            *power-review-api.close_review*
api.close_review({callback?})
  Async. Closes the current review session.

                                          *power-review-api.reply_to_thread*

  Returns: `DraftComment|nil, string|nil`

8. MCP SERVER                                            *power-review-mcp*

The CLI tool doubles as an MCP (Model Context Protocol) server. Running
`powerreview mcp` starts a stdio-based MCP server that AI agents can
connect to directly -- no Neovim instance required.

Configure your AI tool's MCP settings: >json
  {
    "mcpServers": {
      "power-review": {
        "command": "powerreview",
        "args": ["mcp"]
      }
    }
  }
<

MCP tools (22 total):

  Read-only:
    GetReviewSession, ListChangedFiles, GetFileDiff,
    ListCommentThreads, GetDraftCounts, GetPullRequestDescription

  Sync & iteration:
    SyncThreads, CheckIteration, GetIterationDiff

  Write (drafts):
    CreateComment, ReplyToThread, EditDraftComment, DeleteDraftComment,
    UpdateThreadStatus

  File access:
    GetWorkingDirectory, ReadFile, ListRepositoryFiles

  Fix worktree:
    PrepareFixWorktree, GetFixWorktreePath, CreateFixBranch

  Proposals:
    CreateProposal, ListProposals, GetProposalDiff

AI-created comments are tagged with author=ai and start as drafts
that require user approval before submission.

Proposals represent code fixes on temporary branches. The user must
approve and apply them before changes are merged into the PR branch.
Approve/apply/reject are user-only operations (not exposed as MCP tools).

See `skills/reviewing-prs/references/TOOLS.md` for complete API docs.
See `skills/responding-to-comments/SKILL.md` for the comment response workflow.

10. DRAFT WORKFLOW                                    *power-review-drafts*

Comments follow a strict lifecycle: draft -> pending -> submitted.

  draft       Created locally. Editable/deletable. LLMs can modify.
  pending     Approved, ready to submit. Immutable to LLMs.
  submitted   Sent to remote. Immutable.

This ensures LLM-generated comments always go through human approval.

11. SESSION STATE FILE                            *power-review-state-file*

Each review session is persisted as a JSON file on disk by the CLI tool.

FILE LOCATION ~
                                                  *power-review-state-path*
Session files are managed by the CLI and stored at:
>
  {data_dir}/sessions/{session_id}.json
<
The data directory is resolved in this order:
1. `data_dir` field in CLI's `powerreview.json`
2. `$XDG_DATA_HOME/PowerReview`
3. Windows: `%LOCALAPPDATA%\PowerReview`; Linux/Mac: `~/.local/share/PowerReview`

SCHEMA OVERVIEW ~
                                                *power-review-state-schema*
The CLI outputs v5 JSON format with a nested structure (pull_request,
provider, git, review, proposals groupings). The full schema is documented
in `doc/session-schema.md`.

The Neovim plugin adapter converts v4 to a flat Lua shape:
  - `pull_request.id` -> `pr_id`
  - `pull_request.title` -> `pr_title`
  - `provider.type` -> `provider_type`
  - `git.worktree_path` -> `worktree_path`
  - `threads.items` -> `threads` (array)
  - `drafts` map `{uuid: draft}` -> `drafts` (sorted array)
  - `vote` string enum -> numeric value
  - `review.reviewed_files` -> `reviewed_files`
  - `review.changed_since_review` -> `changed_since_review`

See `lua/power-review/cli.lua:adapt_session()` for the full mapping.

PERSISTENCE BEHAVIOR ~
                                          *power-review-state-persistence*

The CLI re-saves the session to disk after every state-changing operation:

  - Adding, editing, or deleting a draft comment
  - Approving or unapproving a draft
  - Submitting pending comments
  - Setting a vote
  - Syncing remote threads
  - Opening, refreshing, or closing a session
  - Marking files as reviewed
  - Checking iterations
  - Creating, approving, applying, rejecting, or deleting proposals
  - Preparing or cleaning up fix worktrees

12. STATUSLINE                                      *power-review-statusline*

The statusline module provides integration for lualine and other
statusline plugins.

Usage with lualine: >lua
  require("lualine").setup({
    sections = {
      lualine_x = {
        require("power-review.statusline").lualine(),
      },
    },
  })
<

Manual usage: >lua
  local sl = require("power-review.statusline")
  if sl.is_active() then
    print(sl.get())
  end
<

The statusline shows: review icon, PR number, review progress,
draft counts, and per-file comment counts (active/resolved/drafts).

13. FILE WATCHER                                    *power-review-watcher*

When a review session is active, the plugin watches the session JSON file
for external changes using UV fs_event. This enables real-time updates
when an AI agent modifies the session via the MCP server.

The watcher is debounced (default 200ms) and automatically reloads the
session, refreshes all UI components, and notifies about AI draft changes.

Configure via `watcher.enabled` and `watcher.debounce_ms`.

14. NOTIFICATIONS                                *power-review-notifications*

PowerReview provides categorized, toggleable notifications:

  ai_activity    -- AI draft created/edited/deleted
  sync_complete  -- Thread sync finished
  watcher        -- External session file changes detected

Toggle at runtime: `:PowerReview toggle_notifications`

Configure individual categories via `notifications.*` in setup.
